# Role Onboarding Template

Use this checklist when a new person joins a project in any role. Complete it during or immediately after the project kickoff meeting. Adapt it to your team's tools and processes.

See also: [Roles & Personas](octoacme-roles-and-personas.md) | [Responsibility Matrix](roles-and-personas-matrix.md)

## Access & Tools

- [ ] Access granted to project board (GitHub Projects / Jira / other)
- [ ] Access granted to source code repository (GitHub / GitLab / other)
- [ ] Added to relevant Slack / Teams channels
- [ ] Access to shared documentation (Confluence / Notion / SharePoint / other)
- [ ] CI/CD and deployment tooling access confirmed (if applicable)
- [ ] Design tool access confirmed (Figma / other — if UX/design role)
- [ ] Monitoring and observability tool access confirmed (if DevOps / on-call role)

---

## People & Stakeholders

- [ ] Stakeholder list reviewed and understood
- [ ] Key contacts for each role identified (use the [Roles & Personas doc](octoacme-roles-and-personas.md))
- [ ] Escalation path documented: who to contact for blockers, decisions, or urgent issues
- [ ] External contacts or vendor contacts documented (if applicable)

---

## Key Artifacts

- [ ] Project one-pager / charter reviewed
- [ ] Roadmap and release plan reviewed
- [ ] Current backlog and prioritization reviewed
- [ ] Definition of Done (DoD) reviewed
- [ ] Risk register reviewed
- [ ] Architecture or technical design docs reviewed (if relevant to role)
- [ ] Responsibility matrix reviewed and role confirmed: [Responsibility Matrix](roles-and-personas-matrix.md)

---

## Recurring Meetings

- [ ] Added to daily standup invite
- [ ] Added to sprint planning invite
- [ ] Added to sprint review / demo invite
- [ ] Added to retrospective invite
- [ ] Added to weekly PM / stakeholder sync invite (if applicable)
- [ ] Calendar holds for any milestone reviews or release dates added

---

## Role-Specific Expectations

- [ ] Success metrics for this role documented and understood
- [ ] Preferred communication channels agreed with the team (async vs. sync, channels for decisions vs. FYIs)
- [ ] Responsibilities confirmed using the RACI matrix
- [ ] Handoff points with adjacent roles understood (e.g., BA → Dev handoff, Dev → QA handoff, DevOps release steps)
- [ ] Code of conduct / team norms reviewed and acknowledged
- [ ] Working hours and availability communicated to the team

---

## First-Week Goals

- [ ] Introductions completed with core team members
- [ ] First contribution or deliverable identified (story, design review, test plan, etc.)
- [ ] Any blockers or missing access reported to PM or Scrum Master
- [ ] Questions or concerns documented and raised in the next standup or 1:1
